# 后端 P7 项目面试 — 面试官评分与追问手册

本文件供 `project-mock-interview` skill 在备课、点评、出题时按需加载。不面向候选人全文背诵。

---

## 1. P7 项目叙述期望（总览）

| 维度 | P6 常见差距 | P7 期望 |
|------|-------------|---------|
| **角色边界** | 「我们做了…」 | 能说清**你**负责的决策、协作接口、不做什么 |
| **问题定义** | 只讲方案 | 业务/技术问题、约束、成功标准先说清 |
| **技术深度** | 堆名词 | 关键路径、瓶颈、取舍（CAP、一致性、成本）说得通 |
| **数据与结果** | 模糊「提升很多」 | 指标口径、基线、观测方式、是否可归因 |
| **复盘** | 只讲成功 | 失败/延期/回滚/技术债与若重来 |
| **沟通** | 发散 | 2～3 分钟能讲完主线，追问时能下钻 |

## 2. 评分维度（C2 点评用）

每题点评可从下列维度选 **1～2 个最相关** 的写缺口，避免面面俱到流于空泛。

### 2.1 可信度（Credibility）

- 时间线、团队规模、技术栈是否与材料一致
- 数字是否能说清来源（监控、AB、对账）
- 难点是否具体（非「高并发」四字）

**探针问题示例**

- 这个 QPS/延迟数字是谁、在什么环境、哪段时期量的？
- 如果线上再发生一次，你第一步看哪个面板/日志？

### 2.2 深度（Depth）

- 是否讲清「为什么选 A 不选 B」
- 是否触及失败模式、降级、幂等、数据一致性
- 能否画简化架构（3～6 个盒子）

**探针问题示例**

- 当时还评估过哪些方案？否决理由？
- 最痛的一次线上问题是什么？怎么修的？

### 2.3 结构（Structure）

- 背景 → 目标 → 行动 → 结果 是否完整
- 先结论后细节，还是绕圈

**探针问题示例**

- 用 30 秒说清：你在这个项目里**唯一**负责的最硬一块是什么？

### 2.4 影响（Impact）

- 业务/成本/稳定性哪条主线
- 结果是否可验证、可复用

**探针问题示例**

- 如果砍掉你这个模块，业务/系统会怎样？

### 2.5 协作与推动力（Influence）

- 跨团队对齐、排期、争议处理
- 技术方案如何落地到规范/平台

**探针问题示例**

- 谁反对过你的方案？你怎么说服或妥协的？

## 3. 项目深挖 — 追问话术库（每次只选 1 条出题）

### 3.1 开场与定位

- 用 2 分钟介绍这个项目：业务背景、你的角色、最终交付。
- 团队里还有谁？你和上游/下游的接口是什么？

### 3.2 目标与约束

- 当时要解决的核心矛盾是什么？不做会怎样？
- 有哪些硬约束（时间、人力、合规、遗留系统）？

### 3.3 架构与决策

- 画一下核心链路（请求从哪来到哪去）。
- 这个技术选型当时最大的争议点是什么？
- 如果流量再涨 10 倍，你先改哪一层？

### 3.4 难点与落地

- 最难的不是写代码，而是哪件事？
- 有没有做过灰度/回滚？策略是什么？

### 3.5 指标与结果

- 成功指标是什么？上线前后对比？
- 有没有误报/副作用？怎么发现的？

### 3.6 复盘与诚信

- 如果重来，你会改哪一个决策？为什么？
- （针对材料模糊处）你刚说的 X，和文档里 Y 不一致，以哪个为准？能举个具体例子吗？

## 4. 红旗信号（备课「高风险点」）

出现以下情况应优先列入备课清单，并在合适轮次追问：

- 简历/文档指标无口径、无基线
- 「负责」与「参与」混用
- 技术栈罗列但与描述场景不匹配
- 难点全是通用词（高并发、微服务）无实例
- 结果只有定性没有定量
- 时间跨度与职责量明显不匹配

## 5. C2 点评写法速查

| 区块 | 要求 |
|------|------|
| 亮点 | 引用用户原话或材料中的具体点，1～2 条 |
| 缺口 | 对照本节维度，bullet，可操作（「应补充 XX 数字」） |
| 参考结构 | 只给提纲，如「30s：背景｜2min：决策链｜1min：结果+复盘」 |

禁止：生成 200 字以上「标准答案」供背诵。

---

## 6. Debrief 总评维度

结束时对整场覆盖：

1. **主线清晰度** — 能否让面试官 3 分钟建立心智模型  
2. **下钻能力** — 追问时是否越答越具体  
3. **可信度** — 数字、角色、时间线是否经得住探针  
4. **P7 差距** — 缺的是深度、影响力还是复盘  
5. **材料行动项** — 回写简历/设计 doc 的最小补充清单
