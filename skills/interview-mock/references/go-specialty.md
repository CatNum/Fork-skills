# Go 语言专项

用户选择 `A. Go 语言专项` 后读取本文件。

## 面试目标

考察候选人是否能超越 API 使用层面理解 Go：运行时行为、并发安全、性能、可观测性和生产问题排查。

## 重点方向

- 运行时与调度器：G-M-P 模型、goroutine 调度、栈增长、系统调用阻塞、抢占、work stealing（工作窃取）。
- 内存与 GC：逃逸分析、堆与栈、写屏障、三色标记、GC pacing（垃圾回收节奏控制）、延迟取舍。
- 并发：channel 语义、select 行为、context 取消、WaitGroup、Mutex/RWMutex、sync.Once、sync.Map、atomic 原子操作。
- 数据结构：map 内部结构、slice 扩容、string/byte 转换成本、interface 表示、nil interface 陷阱。
- 错误与韧性：错误包装、panic/recover 边界、超时传播、goroutine 泄漏预防。
- 性能优化：pprof、trace、benchmark 设计、减少分配、锁竞争、热点路径优化。
- 生产场景：服务延迟尖刺、内存增长、死锁、数据竞争、背压、优雅关闭。

## 提问方式

优先问场景题，例如线上出现某个现象，让候选人判断、定位和处理。少问纯定义题。

不要说“可以从调度、GC、锁竞争几个方面回答”这类提示，让候选人自己选择切入口。

## 题型模式适配

按用户选择的题型模式出题，并遵循 `references/question-taxonomy.md`：

- **实战题模式**：围绕线上延迟尖刺、内存增长、goroutine 泄漏、数据竞争、死锁、GC 抖动等场景提问。
- **八股题模式**：围绕 G-M-P、GC、channel、slice、map、interface、context、锁等概念和机制提问。
- **混合模式**：先问实战题；如果机制解释不清，再追窄机制题；基础薄弱时插入八股题。

## P7 标准

强回答应能连接：

- 语言机制。
- runtime（运行时）实现。
- 生产现象。
- 诊断方法。
- 取舍或预防策略。

## 追问策略

- 如果回答只有概念，追问真实排查路径。
- 如果回答只给工具，追问每个工具能证明什么信号。
- 如果回答忽略取舍，追问修复后可能引入什么问题。
- 如果回答缺少 owner 意识，追问如何防止再次发生。
