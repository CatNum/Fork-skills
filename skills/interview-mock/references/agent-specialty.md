# Agent 专项

用户选择 `F. Agent 专项` 后读取本文件。

## 面试目标

考察候选人是否能设计、评估和运维真正可上线的 AI Agent 系统，而不是只停留在 demo：包括工具调用、规划、记忆、安全、评估、成本、延迟和生产故障处理。

## 重点方向

- Agent 架构：planner/executor（规划器/执行器）、router（路由器）、tool registry（工具注册表）、状态机、固定 workflow（工作流）与自主循环的边界。
- 工具调用：schema（参数结构）设计、权限边界、参数校验、重试、幂等、副作用控制。
- 上下文与记忆：检索、摘要、长上下文管理、用户记忆、会话记忆、记忆新鲜度。
- 终止与控制：停止条件、最大迭代轮数、循环检测、人工审批、失败兜底。
- 评估体系：任务成功率、golden set（黄金样本集）、回归测试、judge model（评测模型）、人工复核。
- 安全边界：prompt injection（提示词注入）、tool injection（工具注入）、数据外泄、密钥处理、沙箱、审计日志。
- 可靠性：部分失败、幻觉工具调用、过期上下文、限流、降级响应。
- 产品适配：用户意图、可解释性、信任建立、升级人工路径、成本和延迟预算。
- 多 Agent 设计：角色拆分、共享状态、冲突解决、编排成本。

## 提问方式

从真实 Agent 产品场景切入，例如工具误用、无限循环、错误记忆、检索失败、不安全动作、评估漂移、成本暴涨、延迟退化。

问题里不要暴露答题分类，也不要提示用户应该从哪些方向回答。

## P7 标准

强回答应体现：

- 明确的控制边界。
- 生产级评估体系。
- 安全与权限推理能力。
- 可度量的可靠性策略。
- 用户体验和系统成本之间的取舍。

## 追问策略

- 如果回答只描述流程，追问系统如何停止。
- 如果回答忽略安全，追问 prompt injection 如何一路影响到工具调用。
- 如果回答只说“加 RAG”，追问检索质量如何评估。
- 如果回答缺少生产约束，追问如何控制延迟和成本。
