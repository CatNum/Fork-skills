# MySQL 专项

用户选择 `B. MySQL 专项` 后读取本文件。

## 面试目标

考察候选人是否把 MySQL 当成生产系统理解：事务语义、锁、索引、SQL 执行、复制、表结构演进和性能故障处理。

## 重点方向

- 索引：B+Tree 结构、聚簇索引、二级索引、覆盖索引、最左前缀、索引条件下推、基数、索引失效。
- SQL 执行：优化器选择、EXPLAIN、filesort、临时表、join 顺序、深分页、聚合、子查询改写。
- 事务：ACID、隔离级别、MVCC、undo log、read view、当前读与快照读。
- 锁：record lock（记录锁）、gap lock（间隙锁）、next-key lock（临键锁）、意向锁、metadata lock（元数据锁）、死锁检测。
- 日志与持久性：redo log、undo log、binlog、两阶段提交、崩溃恢复。
- 复制：异步复制、半同步复制、复制延迟、row/statement/mixed binlog、GTID、读写一致性。
- 运维：慢查询诊断、在线 DDL、大表迁移、备份恢复、容量规划。
- 架构：分库分表、读写分离、幂等、一致性边界。

## 提问方式

从真实问题切入：慢查询、死锁、锁等待、复制延迟、DDL 失败、数据不一致、热点行、大表增长。

不要在问题中包含答题维度。

## 题型模式适配

按用户选择的题型模式出题，并遵循 `references/question-taxonomy.md`：

- **实战题模式**：围绕线上慢查询、死锁、复制延迟、DDL 卡住、数据不一致、大表增长等场景提问。
- **八股题模式**：围绕索引、事务、锁、日志、复制、执行计划、隔离级别等概念和机制提问。
- **混合模式**：先问实战题；如果用户回答中暴露机制漏洞，再追很窄的机制追问题；基础薄弱时插入八股题。
- 用户要求沉淀题目时，按 `references/question-taxonomy.md` 把题型分类写进问题小标题。

示例节奏：

1. 实战题：订单列表分页查询突然变慢，你怎么定位和止血？
2. 机制追问题：`LIMIT 10000,20` 前面被跳过的 10000 条到底会不会回表？你怎么判断？
3. 八股题：什么是覆盖索引？它为什么能避免回表？

## P7 标准

强回答应体现：

- 推断可能机制的能力。
- 准确的验证步骤。
- 安全的止血顺序。
- 长期预防方案。
- 对业务一致性影响的意识。

## 追问策略

- 如果回答说“加索引”，追问为什么这个索引有效，以及如何证明。
- 如果回答说“分库分表”，追问为什么更轻的方案不够。
- 如果回答提到锁，追问具体是什么锁、由哪条语句持有。
- 如果回答提到复制延迟，追问如何保证单个用户读己之写。
