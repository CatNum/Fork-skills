# 面试题学习资料模板

> 写入学习资料时，必须用 Markdown 加粗标记突出重点内容，例如 **核心结论**、**关键术语**、**机制名称**、**关键步骤**、**风险边界**、**取舍判断**、**常见误区** 和 **追问考点**。不要整段加粗，只加粗需要背诵或快速扫读的关键词、短语或句子。

> 学习资料写入由后台子 Agent 执行，避免影响主线程继续模拟面试。子 Agent 只负责生成和追加学习资料；主线程应继续一问一答、点评和追问节奏。

> 每次沉淀新问题时，只能在已有学习资料文档末尾追加一个新的完整问答条目。不要修改、重写、合并、排序或补全已有内容。由已有问题追问出来的新问题，也必须作为新的独立问答继续追加。

## 一句话结论

用 1 到 3 句话给出面试时可以先说出口的核心结论。

## 正确回答结构

### 1. 背景与问题本质

说明这个问题真正考察什么，以及它在生产环境或工程实践中的意义。

### 2. 核心机制

解释必须掌握的原理、关键概念、运行机制或底层实现。

### 3. 处理思路

按面试表达顺序写清楚应该如何分析、设计、排查或优化。

### 4. 风险与取舍

说明不同方案的适用场景、代价、边界和容易踩坑的地方。

## 标准答案

写成可以在面试中直接口头复述的版本，强调结论、依据、步骤和取舍。

## 可背诵版本

控制在短段落或要点形式，用于面试前快速复习。
